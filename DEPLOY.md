# Deploy Fishmaster

Production stack:

| Service | Host | Cost |
|---------|------|------|
| **API** + PostgreSQL | [Render](https://render.com) | Free tier |
| **Web** | [Vercel](https://vercel.com) | Free tier |
| **Mobile** | Expo Go (dev) / EAS (later) | — |

Estimated time: **30–45 minutes**.

## Part 2 — Deploy API + database on Render

### Step 1: Create Render account

1. Go to https://render.com and sign up with GitHub.
2. Authorize access to your `olanr` repository.

### Step 2: Deploy with Blueprint

1. Dashboard → **Blueprints** → **New Blueprint Instance**.
2. Connect repo: `olanrewajuayooluwa717-sys/olanr`.
3. Render reads `render.yaml` and creates:
   - **fishmaster-api** (web service)
   - **fishmaster-db** (PostgreSQL)
4. Click **Apply**.

Wait ~5–10 minutes for the first deploy.

### Step 3: Copy your API URL

When deploy finishes, open **fishmaster-api** → copy the URL, e.g.:

```
https://fishmaster-api.onrender.com
```

Test in browser: `https://fishmaster-api.onrender.com/health`

You should see: `{"status":"ok","service":"fishmaster-api","database":true}`

### Step 4: Set API environment variables

In **fishmaster-api** → **Environment**:

| Key | Value |
|-----|--------|
| `JWT_SECRET` | Auto-generated by Render (keep it) |
| `WEB_URL` | Leave blank for now — fill after Vercel (Step 3 below) |
| `STRIPE_SECRET_KEY` | Your Stripe test/live secret key |
| `STRIPE_WEBHOOK_SECRET` | From Stripe webhook setup (Part 4) |

`DATABASE_URL` is wired automatically from the Postgres addon.

## Part 3 — Deploy web on Vercel

### Step 1: Create Vercel account

1. Go to https://vercel.com and sign up with GitHub.

### Step 2: Import project

1. **Add New…** → **Project**.
2. Import `olanrewajuayooluwa717-sys/olanr`.
3. **Root Directory:** click Edit → set to `apps/web`.
4. Framework: **Next.js** (auto-detected).

### Step 3: Environment variables

Add before deploying:

| Key | Value |
|-----|--------|
| `NEXT_PUBLIC_API_URL` | `https://fishmaster-api.onrender.com` (your Render API URL) |

### Step 4: Deploy

Click **Deploy**. Wait ~2–3 minutes.

Copy your web URL, e.g. `https://olanr.vercel.app`.

### Step 5: Link API back to web

1. Render → **fishmaster-api** → **Environment**.
2. Set `WEB_URL` = your Vercel URL (e.g. `https://olanr.vercel.app`).
3. **Save** — Render redeploys automatically.

---

## Part 4 — Stripe (subscriptions)

1. [Stripe Dashboard](https://dashboard.stripe.com) → **Developers** → **Webhooks**.
2. **Add endpoint:**
   - URL: `https://fishmaster-api.onrender.com/api/billing/webhook`
   - Events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`
3. Copy **Signing secret** → paste into Render as `STRIPE_WEBHOOK_SECRET`.
4. Add `STRIPE_SECRET_KEY` (test keys for now, live when ready).

## Part 5 — Mobile app (Expo Go)

Update `apps/mobile/.env`:

```env
EXPO_PUBLIC_API_URL=https://fishmaster-api.onrender.com
```

Restart Expo: `npx expo start -c`

> **Note:** Render free tier sleeps after ~15 min idle. First API request may take 30–60 seconds to wake up.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| API health shows `database: false` | Check `DATABASE_URL` on Render; re-run deploy |
| Web "Network request failed" | `NEXT_PUBLIC_API_URL` must match Render API URL exactly |
| Render build fails | Check logs; ensure `npm run build:api` works locally |
| Vercel build fails | Root directory must be `apps/web` |
| Stripe checkout fails | Set `WEB_URL` and `STRIPE_SECRET_KEY` on Render |
| Slow first load | Render free tier cold start — normal |
